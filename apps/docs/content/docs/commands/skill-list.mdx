# /skill:list

Display all skills installed in `.claude/skills/` with their metadata and file information.

## Quick Start

```bash
/agileflow:skill:list
```

Shows all installed skills with descriptions, file counts, and MCP status.

## What It Does

Scans your installed skills and displays:

- **Skill names** - Identifier from SKILL.md
- **Descriptions** - Purpose and use cases
- **File count** - How many files in each skill
- **Cookbook entries** - Use-case workflows included
- **MCP status** - Whether MCP is configured
- **Action recommendations** - What to do next

## Parameters

This command has no parameters.

```bash
/agileflow:skill:list
```

## Understanding the Output

### Skill Entry

Each skill shows:

1. **Name** - Identifier for the skill (e.g., `supabase-swift`)
2. **Description** - From the skill's frontmatter
3. **Files** - Breakdown of included files:
   - `SKILL.md` - Main skill file (always present)
   - `references.md` - Documentation links (usually present)
   - `cookbook/*.md` - Use-case workflows
   - `.mcp.json` - MCP server config (if applicable)
4. **MCP Status** - "âœ“" if `.mcp.json` exists and is configured

### File Information

The file count includes:

- **SKILL.md** - 1 file
- **references.md** - 1 file (if exists)
- **cookbook/** - Multiple files (one per use case)
- **.mcp.json** - 1 file (if configured)

Example breakdown:
```
5 files total:
  - SKILL.md (main)
  - references.md (links)
  - cookbook/crud-operations.md (workflow)
  - cookbook/authentication.md (workflow)
  - .mcp.json (MCP config)
```

## Best Practices

### Organizing Skills

Group related skills by domain:

- **Data layer**: `supabase-swift`, `postgresql-queries`, `mongodb-patterns`
- **API layer**: `rest-api-errors`, `graphql-schema`, `webhooks`
- **UI layer**: `react-components`, `accessibility-wcag`, `tailwind-patterns`
- **DevOps**: `github-actions`, `docker-deployment`, `kubernetes`

### Regular Audits

Periodically run `/skill:list` to:

1. Check for unused skills
2. Verify MCP configuration
3. Identify skills that need updating
4. See total skill count

### Cleaning Up

Delete skills you no longer use:

```bash
/agileflow:skill:delete old-skill-name
```

This keeps your skills directory focused and easier to manage.

## Troubleshooting

### Skills Directory Missing

If you see "No skills installed yet" and expected skills:

1. Check if `.claude/skills/` exists:
   ```bash
   ls -la .claude/skills/
   ```

2. Verify SKILL.md files exist in subdirectories:
   ```bash
   find .claude/skills -name "SKILL.md"
   ```

3. Ensure proper read permissions on the directory

### Incomplete Skill Information

If a skill shows incomplete metadata:

1. The SKILL.md file may have corrupt frontmatter
2. Run `/agileflow:skill:test` to diagnose
3. Edit with `/agileflow:skill:edit` to fix

### MCP Not Showing

If you expect MCP but don't see "âœ“ Configured":

1. Check if `.mcp.json` exists:
   ```bash
   ls .claude/skills/<skill-name>/.mcp.json
   ```

2. Verify it has valid JSON syntax
3. Upgrade or edit the skill to fix MCP configuration
