# /agileflow:skill:list

Display all skills installed in `.claude/skills/` with their metadata and file structure.

## Quick Start

```bash
/agileflow:skill:list
```

## How It Works

This command scans your `.claude/skills/` directory and displays:

1. **Skill names and descriptions** - From SKILL.md frontmatter
2. **File counts** - How many supporting files each skill has
3. **Cookbook entries** - Available workflows for each skill
4. **MCP status** - Whether a skill has MCP integration
5. **Next action prompts** - Quick access to other skill commands

## What's Displayed

For each skill, you'll see:

| Item | Description |
|------|-------------|
| **Skill Name** | Directory name from `.claude/skills/` |
| **Description** | From `description:` in SKILL.md frontmatter |
| **MCP Status** | Shows "MCP: ‚úì" if `.mcp.json` is present |
| **File Count** | Total number of supporting files |
| **Files Listed** | Cookbook entries and other supporting files |

## How the Command Works

### Step 1: Check for Skills Directory

Checks if `.claude/skills/` exists and contains skill directories:

```bash
ls -la .claude/skills/ 2>/dev/null
```

If the directory doesn't exist or is empty:

```
No skills installed yet.

To create a skill, run: /agileflow:skill:create
```

### Step 2: Scan Each Skill Directory

For each subdirectory in `.claude/skills/`:

1. **Read SKILL.md frontmatter** to extract:
   - `name` - Skill identifier
   - `description` - Purpose of the skill

2. **Count supporting files**:
   - `cookbook/*.md` - Workflow entries
   - `tools/` - Executable scripts
   - `.mcp.json` - MCP configuration
   - `references.md` - Documentation links

3. **Check for MCP integration**:
   - If `.mcp.json` exists, note "MCP: ‚úì"

### Step 3: Format Output

Displays all skills in a formatted table with:
- Skill names and descriptions
- File counts
- Cookbook entry names
- MCP status

### Step 4: Offer Next Actions

After listing, you can:

```
What would you like to do?
  ‚Ä¢ Create new skill
  ‚Ä¢ Edit a skill
  ‚Ä¢ Delete a skill
  ‚Ä¢ Test a skill
  ‚Ä¢ Done
```

## Skill Metadata Format

Each skill's metadata comes from `SKILL.md` frontmatter:

```yaml
---
name: supabase-swift
description: Use when building Swift apps with Supabase database operations
---
```

The command extracts this with:

```bash
head -10 .claude/skills/<skill>/SKILL.md | grep -E "^(name|description):"
```

## Understanding Skill Status

### Complete Skills

Have all recommended files:
- SKILL.md
- references.md
- cookbook/ directory with 2+ entries
- .mcp.json (if applicable)

### Basic Skills

Have minimal setup:
- SKILL.md only
- references.md
- No cookbook or MCP
