# /skill:edit

Modify an existing skill in `.claude/skills/` - update instructions, add cookbook entries, change references, or completely rewrite with fresh research.

## Quick Start

```bash
# Interactive mode
/agileflow:skill:edit

# Edit specific skill
/agileflow:skill:edit supabase-swift
```

## What It Does

Allows you to modify any aspect of a skill:

- **SKILL.md** - Update activation triggers and instructions
- **references.md** - Add or update documentation links
- **Cookbook entries** - Edit specific use-case workflows
- **Add new cookbook** - Create workflows for new use cases
- **MCP configuration** - Update MCP server settings
- **Rename skill** - Change the skill name and folder

## Special Edit Operations

### Full Rewrite (Recommended for Major Changes)

When you select "Full rewrite":

1. **Enter Plan Mode** - Command researches current best practices
2. **Collect new sources**:
   - Updated web documentation
   - New local documentation
   - Current codebase patterns
3. **Show updated plan** - You approve changes
4. **Regenerate** - Creates fresh version with new research
5. **Preserve history** - Old version is backed up

This is ideal when:
- Technology version has major updates
- Your project patterns have evolved
- You want to refresh all content

### Add New Cookbook Entry

When you select "Add new cookbook":

1. **Name the use case** - e.g., "pagination", "bulk-operations"
2. **Choose template** - Cookbook structure is auto-filled
3. **Update SKILL.md** - References are added automatically
4. **Edit if needed** - Refine the generated content

### Update MCP Configuration

When you select "MCP configuration":

1. **View current .mcp.json**
2. **Edit server settings**
3. **Add/remove environment variables**
4. **Test configuration** - Verify syntax

## Best Practices

### When to Edit

Edit a skill when:
- Technology version updates
- Your team discovers new patterns
- New use cases emerge
- Documentation changes
- Errors need fixing

### Keeping Skills Current

1. **Regular audits** - Test skills quarterly
2. **User feedback** - Edit based on what Claude gets wrong
3. **Pattern evolution** - Update as your code patterns evolve
4. **Docs updates** - Keep external doc links current

### Making Effective Changes

1. **Small, focused changes** - Update one thing at a time
2. **Always preview** - Check diffs before saving
3. **Test after editing** - Run `/skill:test` after major changes
4. **Document why** - Add comments explaining changes

## Error Handling

### Skill Not Found

```

  - supabase-swift
  - ui-components

Use /agileflow:skill:list to see all skills.
```

**Fix**: Check the skill name spelling, or list available skills.

### Cannot Parse Frontmatter

```
âš ï¸ Could not parse SKILL.md frontmatter.

Would you like me to try anyway, or edit manually?
```

**Fix**: Check SKILL.md has valid YAML frontmatter:
```yaml
---
name: skill-name
description: Skill purpose
---
```

### Write Permission Error

```
âŒ Cannot write to skill file.

Check permissions for: .claude/skills/<skill>/
```

**Fix**: Verify directory permissions:
```bash
chmod -R u+w .claude/skills/<skill>/
```

## After Editing

Once changes are saved:

1. **Test the skill**:
   ```bash
   /agileflow:skill:test <skill-name>
   ```

2. **Use immediately** - Changes take effect next use

3. **If errors** - Use `/skill:edit` again to fix

4. **Enable learning** - If you haven't already:
   ```bash
   /agileflow:skill:upgrade <skill-name>
   ```
