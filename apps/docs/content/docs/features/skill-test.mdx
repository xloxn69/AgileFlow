# /skill:test

Test a skill to verify it activates correctly, has proper structure, and produces expected results.

## Quick Start

```bash
# Interactive mode
/agileflow:skill:test

# Test specific skill
/agileflow:skill:test supabase-swift
```

## What It Does

The test command validates skills through three phases:

1. **Structure Validation** - Files exist and are properly formatted
2. **Content Validation** - Required sections present, no broken links
3. **Activation Test** (optional) - Verify skill triggers on sample prompts

## Validation Checks

### Structure Checks

| Check | What It Verifies |
|-------|------------------|
| **SKILL.md exists** | Main skill file is present |
| **Frontmatter valid** | Has `name:` and `description:` |
| **references.md** | Documentation links file (recommended) |
| **cookbook/ directory** | Workflows exist (if referenced) |
| **.mcp.json valid** | JSON syntax correct (if present) |

### Content Checks

| Check | What It Verifies |
|-------|------------------|
| **"When to Use" section** | Activation triggers defined |
| **Description length** | Under 1024 characters |
| **SKILL.md size** | Under 500 lines |
| **Cookbook references** | All files in SKILL.md exist |
| **No broken links** | Local file paths are valid |

## Interpreting Results

### Structure Issues

**Problem**: "SKILL.md missing"
- **Cause**: Skill directory doesn't contain SKILL.md
- **Fix**: Verify directory exists: `ls .claude/skills/<name>/SKILL.md`
- **Solution**: Use `/skill:edit` to recreate

**Problem**: "Frontmatter invalid"
- **Cause**: SKILL.md YAML is malformed
- **Fix**: Check first 5 lines have valid YAML:
  ```yaml
  ---
  name: skill-name
  description: Purpose
  ---
  ```
- **Solution**: Use `/skill:edit` to fix frontmatter

**Problem**: "No 'When to Use' section"
- **Cause**: Activation triggers not defined
- **Fix**: Add section to SKILL.md:
  ```markdown
  ## When to Use

  This skill activates when:
  - Condition 1
  - Keywords: "keyword1", "keyword2"
  ```
- **Solution**: Use `/skill:edit` to add section

### Content Issues

**Problem**: "Description exceeds 1024 characters"
- **Cause**: Skill description is too long
- **Fix**: Shorten the description in SKILL.md frontmatter
- **Solution**: Use `/skill:edit` to reduce length

**Problem**: "SKILL.md exceeds 500 lines"
- **Cause**: Skill is too large/complex
- **Fix**: Move content to cookbook/ files
- **Solution**: Use `/skill:edit` to reorganize

**Problem**: "Broken link: cookbook/missing.md referenced but not found"
- **Cause**: Cookbook file reference without actual file
- **Fix**: Either create the file or remove the reference
- **Solution**: Use `/skill:edit` to fix

### MCP Issues

**Problem**: "Invalid JSON in .mcp.json"
- **Cause**: Syntax error in MCP configuration
- **Fix**: Check JSON syntax is valid
- **Solution**: Use `/skill:edit` to fix JSON

**Problem**: "No mcpServers object"
- **Cause**: .mcp.json missing required structure
- **Fix**: Add proper structure:
  ```json
  {
    "mcpServers": {
      "name": { ... }
    }
  }
  ```
- **Solution**: Use `/skill:edit` to update


### Full Test (Structure + Activation)


Tests that skill actually activates on relevant prompts.

### Detailed Test (Structure + Activation + View Content)


Helps understand what the skill does before using it.

## Best Practices

### Test After Creation

Always test a skill right after creating it:

```bash
/agileflow:skill:create my-skill
/agileflow:skill:test my-skill  # â† Right after creation
```

### Test After Editing

Test after making changes:

```bash
/agileflow:skill:edit my-skill
/agileflow:skill:test my-skill  # â† Verify changes worked
```

### Regular Audits

Test all skills periodically:

```bash
/agileflow:skill:list
# For each skill:
/agileflow:skill:test <skill-name>
```

### Test New Triggers

When you add new activation triggers:

```bash
/agileflow:skill:edit my-skill  # Add trigger keywords
/agileflow:skill:test my-skill  # Test with those keywords
```

## Troubleshooting

### Test Hangs

If the test seems to hang:

1. **Wait up to 30 seconds** - Activation testing can take time
2. **Check internet connection** - If testing web sources
3. **Cancel (Ctrl+C)** and try again
4. **Skip activation test** - Try structure-only test

### Inconsistent Results

If test results vary:

1. **Network issues** - Some sources may be temporarily unavailable
2. **Race conditions** - Try testing again
3. **Check file system** - Ensure files are readable:
   ```bash
   ls -la .claude/skills/<skill>/
   ```

### Activation Test Fails

If "Skill triggered correctly" shows as failed:

1. Update activation triggers:
   ```bash
   /agileflow:skill:edit <skill-name>
   # Edit "When to Use" section
   ```

2. Test with different prompts

3. Check if keywords are specific enough
